Detailed Description

template<typename ItemType, typename ItemContainerType, typename Indexer = MutexIndexer, IndexSizeT FixedPoolSize = 0>
class dxpool::Pool< ItemType, ItemContainerType, Indexer, FixedPoolSize >

A fixed size, thread safe data and object pool.

The pool starts with a fixed number of items and it cannot be resized dynamically. Items in the pool are never moved or haved their memory position changed. This means that consumers will always work with references to items, while the pool is responsible for item lifetime.

This means that the type of items in the pool need to be trivially constructible. For the full benefits of cache locality, avoid sharing pools between threads on different cores.

ItemType's destructor is only invoked when the pool is destroyed.

A pool cannot be copied or moved.

Template Parameters
ItemTypetype of the data to be stored
Indexertype of indexer to be used to when retrieving and returning objects. The indexer defines the concurrency behaviour when using the pool in a multi-threaded context
FixedPoolSizesize of the pool, if specified as fixed
